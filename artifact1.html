Design Thinking Process

Empathize +

We interviewed college peers and small business owners to identify pain points in chatbot creation. Users cited technical complexity and lack of easy interfaces as major barriers.

Define +

We defined the problem: How can we enable non-coders to confidently create functional chatbots with minimal technical jargon?

Ideate +

Brainstormed features such as interactive FAQ builder, tone selector, visual flow preview, multilingual templates, and user success checklist.

Prototype +

Implemented core chatbot features on Mizou.com, embedded in-chat documentation, and designed a clean, user-friendly UI mimicking teaching interactions.

Test +

Conducted peer testing and gathered feedback. Refined bot language, added session memory, and integrated visual aids. MRR improved to 0.86 after iterations.

Challenges & Solutions

Challenge Solution Result
Users overwhelmed by technical language Simplified to 8th grade reading level, used analogies User satisfaction rose significantly
