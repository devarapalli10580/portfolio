Artifact Description

Overview:

The featured artifact is a conversational AI assistant – the ChatBox Creator Bot – designed to help users, especially non-programmers, create and deploy custom chatbots without any coding knowledge. The bot was built using Mizou.com, a no-code platform that allows easy prototyping and deployment of chatbot flows.

The chatbot acts as a mentor and guide, enabling users to construct their own bots by walking them through features such as tone selection, FAQ design, visual flow planning, and integration into web platforms. It supports both beginners and small business owners looking to adopt conversational AI solutions.

Problem Solved:

Many individuals with innovative ideas often face a steep learning curve in implementing chatbots due to lack of technical expertise. This AI assistant addresses that issue by simplifying the development process into intuitive conversations. It provides visual instructions, examples, templates, and hands-on walkthroughs to help users build personalized chatbots.

Design Thinking Applied

Empathize:

We conducted interviews with college peers, small business owners, and entrepreneurs to understand their biggest challenges in chatbot development. Most cited technical jargon, interface complexity, and uncertainty about use cases as primary barriers.

Define:

How might we help non-coders build functional chatbots quickly and confidently without writing code?

Ideate:

As a team, we brainstormed 12+ features. Key ideas included:

  • Interactive FAQ builder
  • Tone selector (casual, formal, friendly, etc.)
  • Visual flowchart previews
  • Multilingual templates
  • User success checklist

Prototype:

Using Mizou, we implemented the core features, embedded in-chat documentation, and designed a clean UI that mimics teaching interactions.

Test:

We conducted peer testing sessions. Based on feedback, we revised the bot’s language style, added memory features for session continuity, and integrated visual aids for better guidance. Final Mean Reciprocal Rank (MRR) performance improved to 0.86.

Reflection

Participating in this chatbot development project taught me several vital lessons:

User-Centric Design:

I learned to think beyond features and focus on how users interact with AI tools. Simplifying complexity into human-friendly interfaces was a powerful experience.

Interdisciplinary Collaboration:

I worked closely with peers from different backgrounds – business, tech, and design – which helped refine my communication and teamwork skills. I contributed heavily to the documentation and final presentation, ensuring clarity across technical and non-technical audiences.

Iterative Learning:

Early user feedback was crucial. We thought our initial version was strong, but user testing revealed gaps. Revising the tone, flow, and visuals significantly improved the final product.

This artifact now stands as a strong example of how I solve problems with empathy, design thinking, and AI expertise – skills I intend to apply in future data analytics and AI development roles.

Feedback and Revisions

Feedback Area Initial Issue Action Taken Results of Revision
Complexity of Language Some users were overwhelmed by AI/tech terms. Simplified language to 8th-grade reading level; added examples and analogies. MRR increased from 0.61 → 0.86
Visual Flow Confusion Users struggled with text-heavy, linear conversation flow. Introduced inline flowcharts, added multimedia elements (diagrams/tutorials). User satisfaction rating: 4.9 / 5.0
Lack of Onboarding New users felt lost at the start. Added a “Welcome & Instructions” sequence for onboarding. Most praised features: Easy-to-follow flow, visual previews, and friendly tone
