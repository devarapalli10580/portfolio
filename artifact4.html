Artifact 4: Data Challenge Scenarios – Navigating Data Quality, Bias, and Privacy in Machine Learning

Objective

This artifact demonstrates my ability to identify, analyze, and resolve critical data challenges encountered in real-world machine learning workflows. These challenges include data quality issues, privacy and security concerns, and bias in AI systems. Successfully addressing these problems ensures AI models are accurate, fair, and trustworthy.

Context of the Work

Developed as part of Module 4: Data Challenges in Machine Learning during my Master’s coursework in Artificial Intelligence – Data Analytics at Indiana Wesleyan University, this activity involved interacting with the SchoolAI Data Challenge Scenarios Coach. The coach presented simulated real-world scenarios involving data preparation, privacy compliance, and bias mitigation.

Process and Methodology

Step 1: Understanding the Scenarios

  • Data Quality Issues: Handling missing values, duplicates, and noisy data in a financial dataset.
  • Privacy and Security: Managing sensitive healthcare data in compliance with GDPR and HIPAA.
  • Bias and Fairness: Addressing imbalanced datasets and algorithmic bias in predictive hiring models.

Step 2: Solutions for Each Scenario

Scenario 1: Data Quality

Challenges Identified:

  • Missing values in key attributes.
  • Duplicate records affecting accuracy.
  • Outliers skewing statistical distributions.

Solutions Applied:

  • Imputed missing numerical values with mean, categorical with mode.
  • Removed duplicate entries.
  • Applied Z-score method to identify and remove outliers.
  • Normalized numerical features using Min-Max scaling.
import pandas as pd
import numpy as np
from scipy import stats
from sklearn.preprocessing import MinMaxScaler

data = pd.read_csv('financial_data.csv')
data['income'].fillna(data['income'].mean(), inplace=True)
data['occupation'].fillna(data['occupation'].mode()[0], inplace=True)
data.drop_duplicates(inplace=True)

z_scores = np.abs(stats.zscore(data.select_dtypes(include=[np.number])))
data = data[(z_scores < 3).all(axis=1)]

scaler = MinMaxScaler()
data[['income', 'age']] = scaler.fit_transform(data[['income', 'age']])
Scenario 2: Privacy and Security

Challenges Identified:

  • Handling sensitive patient data requiring legal compliance.
  • Preventing data breaches during storage and transmission.

Solutions Applied:

  • Removed personal identifiers through anonymization.
  • Encrypted data using AES-256 encryption.
  • Implemented role-based access control.
  • Ensured compliance with GDPR, HIPAA, and FERPA standards.
from cryptography.fernet import Fernet

key = Fernet.generate_key()
cipher_suite = Fernet(key)

sensitive_data = b"Patient PHI data to encrypt"
encrypted_data = cipher_suite.encrypt(sensitive_data)

decrypted_data = cipher_suite.decrypt(encrypted_data)
print(decrypted_data.decode())
Scenario 3: Bias and Fairness

Challenges Identified:

  • Class imbalance in hiring prediction dataset.
  • Gender and ethnicity-based bias risk.

Solutions Applied:

  • Used SMOTE for synthetic minority oversampling.
  • Audited fairness metrics including Equal Opportunity Difference and Demographic Parity.
  • Adjusted model thresholds to reduce disparate impact.
  • Promoted inclusion of diverse stakeholders in model design.
from imblearn.over_sampling import SMOTE
from collections import Counter

X = data.drop('target', axis=1)
y = data['target']

print('Original dataset shape %s' % Counter(y))

sm = SMOTE(random_state=42)
X_resampled, y_resampled = sm.fit_resample(X, y)

print('Resampled dataset shape %s' % Counter(y_resampled))
