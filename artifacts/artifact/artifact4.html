Scenario 1: Data Quality Challenges

Problems identified:

  • Missing values in numeric columns.
  • Duplicate records skewing analytics.
  • Outliers in financial transactions causing model distortion.

Solutions Applied:

  1. Imputation for missing data using mean for numerical and mode for categorical.
  2. Deduplication using Pandas drop_duplicates().
  3. Outlier detection via Z-score and IQR methods.
import pandas as pd
import numpy as np

# Handle missing data
df['age'].fillna(df['age'].mean(), inplace=True)

# Remove duplicates
df.drop_duplicates(inplace=True)

# Outlier detection
from scipy import stats
df = df[(np.abs(stats.zscore(df['salary'])) < 3)]

Scenario 2: Privacy & Security

Challenges:

  • PHI compliance under HIPAA/GDPR.
  • Secure transmission and storage of healthcare records.

Solutions: AES encryption for storage, anonymization for data sharing.

from cryptography.fernet import Fernet

# Generate key and encrypt

Reflection

This exercise reinforced the significance of data integrity, ethical AI, and compliance in machine learning workflows. Key lessons:

  • Garbage in, garbage out – Data quality directly impacts performance.
  • Bias can emerge at multiple stages of ML pipeline.
  • Privacy compliance builds trust and prevents legal issues.
