Artifact 3: SchoolAI-Assisted Learning Experience

This artifact captures my experience engaging with AI-powered educational tools on the SchoolAI platform. The interactive chatbot guided me through data challenge scenarios, reinforcing concepts in machine learning data preparation, privacy, and security.

This learning experience enhanced my practical understanding of AI application challenges while developing critical thinking and problem-solving skills through iterative AI interaction.

Tools & Technologies

  • SchoolAI Chatbot: An AI coach simulating real-world machine learning challenges.
  • Web-based Interactive UI: Seamless chatbot interaction and scenario navigation.
  • JavaScript & HTML5: Custom scripting and structuring of the learning interface.
  • Data Science Concepts: Applied data cleaning, bias mitigation, privacy compliance.

Learning Process

My engagement with the SchoolAI platform involved three key scenario exercises focused on:

  1. Data Cleaning Challenges: Identifying and correcting data quality issues, handling missing data, and outlier detection.
  2. Privacy & Security: Understanding ethical data sharing, anonymization techniques, and GDPR compliance.
  3. Bias Identification: Recognizing human and systemic bias in datasets and proposing mitigation strategies.

Each scenario required making decisions supported by reasoning, receiving immediate AI feedback, and iteratively refining approaches. This method encouraged experimentation, deeper understanding, and confidence-building.

Sample Python Code Snippet: Data Cleaning

import pandas as pd
import numpy as np

# Load dataset
data = pd.read_csv('dataset.csv')

# Handling missing values by imputation
data['Age'].fillna(data['Age'].median(), inplace=True)

# Removing outliers using Z-score
from scipy import stats
z_scores = np.abs(stats.zscore(data.select_dtypes(include=[np.number])))
data = data[(z_scores < 3).all(axis=1)]

# Verify cleaned data
print(data.info())

Challenges & Solutions

  • Challenge: Handling incomplete or corrupted data without biasing models.
    Solution: Used median imputation for missing numeric data and domain expert consultation to verify data integrity.
  • Challenge: Ensuring privacy compliance while sharing data.
    Solution: Applied anonymization and synthetic data generation to protect personally identifiable information.
  • Challenge: Detecting bias in training data.
    Solution: Incorporated bias detection frameworks and adjusted sample weights to balance datasets.
